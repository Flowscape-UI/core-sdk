# üß© Plugins

Plugins extend Flowscape SDK functionality. The SDK includes built-in plugins and supports custom development.

## GridPlugin

Adaptive grid with snap-to-grid functionality.

```typescript
import { GridPlugin } from '@flowscape-ui/core-sdk';

const grid = new GridPlugin({
  enabled: true,
  color: '#e5e7eb',
  cellSize: 20,
  enableSnap: true,
});
```

**Options:**

- `enabled` ‚Äî Grid visibility
- `color` ‚Äî Grid line color
- `cellSize` ‚Äî Cell size in pixels
- `enableSnap` ‚Äî Enable snap-to-grid
- `snapThreshold` ‚Äî Snap distance threshold

## SelectionPlugin

Selection, transformation, drag & drop, and grouping.

```typescript
import { SelectionPlugin } from '@flowscape-ui/core-sdk';

const selection = new SelectionPlugin({
  dragEnabled: true,
  rotateEnabled: true,
  keepRatio: false,

  // DOM video controls overlay for selected VideoNode
  // enableVideoOverlay: true,
  enableVideoOverlay: {
    uiAccentColor: '#ff8a00',
    uiTrackFilledColor: '#ff8a00',
    uiBackgroundColor: 'rgba(18,18,18,0.92)',
  },
});
```

**Features:**

- Multi-selection (Shift+Click)
- Drag & drop
- Rotation handles
- Custom rotation cursor (improved UX during rotation)
- Resize with aspect ratio lock (Shift)
- Grouping (Ctrl+G)
- Video overlay controls (optional)

## ContentFromClipboardPlugin

Paste and drop external content directly into the canvas.

Supports:

- Images (any browser-supported formats)
- GIFs
- Videos (any browser-supported formats)
- SVG files
- Text
- Drag & drop from desktop (optional)

```typescript
import { ContentFromClipboardPlugin } from '@flowscape-ui/core-sdk';

const contentFromClipboard = new ContentFromClipboardPlugin({
  // Where to listen for paste events
  // target: window,

  // Ignore paste events from inputs/textarea/contenteditable
  ignoreEditableTargets: true,

  // Fit pasted media into max size (in px)
  maxImageSize: 1200,

  // Enable drag & drop of files onto the canvas container
  enableDragDrop: true,
});
```

**Notes:**

- Pasting uses the current pointer position if it is on the canvas; otherwise it falls back to the screen center.
- Drag & drop places content at the actual drop point (world coordinates).

**Shortcuts:**

- `Ctrl+C` ‚Äî Copy selected nodes
- `Ctrl+X` ‚Äî Cut selected nodes
- `Ctrl+V` ‚Äî Paste nodes
- `Delete` ‚Äî Delete selected nodes
- `Ctrl+]` ‚Äî Move up (z-index)
- `Ctrl+[` ‚Äî Move down (z-index)

## PersistencePlugin

Auto-save canvas state to IndexedDB with full media support.

```typescript
import { PersistencePlugin } from '@flowscape-ui/core-sdk';

const persistence = new PersistencePlugin({
  canvasId: 'my-canvas',
  debounceMs: 500,
  autoRestore: true,
  debug: false,
});

core.plugins.add(persistence);
```

**Options:**

- `canvasId` ‚Äî Unique identifier for this canvas (default: `'default'`)
- `debounceMs` ‚Äî Delay before auto-save triggers (default: `300`)
- `autoRestore` ‚Äî Automatically restore state on attach (default: `true`)
- `dbName` ‚Äî Custom IndexedDB database name
- `debug` ‚Äî Enable debug logging

**Features:**

- Automatic save on any canvas change (nodes, camera, transforms)
- Full support for media nodes (Image, Video, GIF, SVG)
- Blob URL persistence ‚Äî media files are stored in IndexedDB
- Preserves node dimensions, rotation, and z-index
- Supports GroupNode and FrameNode with nested children
- Export/import canvas as JSON with embedded blobs
- Debounced saves to optimize performance

**Methods:**

```typescript
// Manual save
await persistence.save();

// Restore from IndexedDB
await persistence.restore();

// Clear saved state
await persistence.clear();

// Check if saved state exists
const hasSaved = await persistence.hasSavedState();

// Export canvas to JSON string (with embedded blobs)
const json = await persistence.exportToJSON();

// Import canvas from JSON string
persistence.importFromJSON(json);

// Download canvas as JSON file
await persistence.downloadJSON('my-canvas.json');

// Upload and import from JSON file (opens file picker)
await persistence.uploadJSON();

// Get/set canvas ID
const id = persistence.getCanvasId();
await persistence.setCanvasId('new-canvas-id');
```

**Events Triggering Auto-Save:**

- `node:created`, `node:removed`, `node:transformed`
- `node:zIndexChanged`
- `group:created`, `group:ungrouped`
- `clipboard:paste`
- `camera:zoom`, `camera:pan`, `camera:reset`
- Konva `dragend`, `transformend`

**Storage Structure:**

The plugin uses IndexedDB with two object stores:

- `canvas` ‚Äî Serialized canvas state (JSON)
- `blobs` ‚Äî Binary media files (images, videos, etc.)

**Example: Manual Export/Import**

```typescript
// Export current canvas
const json = await persistence.exportToJSON();
localStorage.setItem('canvas-backup', json);

// Import from backup
const backup = localStorage.getItem('canvas-backup');
if (backup) {
  persistence.importFromJSON(backup);
}
```

## Creating Custom Plugins

```typescript
import { Plugin, CoreEngine } from '@flowscape-ui/core-sdk';

class CustomPlugin extends Plugin {
  protected onAttach(core: CoreEngine): void {
    // Initialize plugin
    console.log('Plugin attached');

    // Subscribe to events
    core.eventBus.on('node:created', this.handleNodeCreated);
  }

  protected onDetach(core: CoreEngine): void {
    // Cleanup
    core.eventBus.off('node:created', this.handleNodeCreated);
  }

  private handleNodeCreated = (node: any) => {
    console.log('Node created:', node.getId());
  };
}

// Usage
const custom = new CustomPlugin();
core.plugins.addPlugins([custom]);

// Later at runtime
core.plugins.removePlugins([custom]);
```

## Plugin Lifecycle

1. **Creation** ‚Äî Plugin instance created
2. **Attach** ‚Äî `onAttach()` called when added to engine
3. **Active** ‚Äî Plugin is active and listening to events
4. **Detach** ‚Äî `onDetach()` called when removed
5. **Cleanup** ‚Äî Resources freed

## Best Practices

1. **Clean up resources** ‚Äî Always unsubscribe in `onDetach()`
2. **Use event bus** ‚Äî Communicate via events
3. **Keep plugins focused** ‚Äî One responsibility per plugin
4. **Document options** ‚Äî Provide clear configuration
5. **Handle errors** ‚Äî Graceful error handling
